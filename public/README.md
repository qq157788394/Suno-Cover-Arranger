# Suno翻唱编曲大师 / Suno Cover Arranger 

## 工具简介

Suno翻唱编曲大师是一款专为Suno翻唱创作者设计的AI工具，帮助用户用中文描述音乐风格和编曲想法，自动生成符合Suno官方术语体系的高质量英文提示词。

## 解决的痛点

### 痛点一：Suno 的 Styles 字段写不好

做过几次 Suno 翻唱的人，基本都会遇到：

* 知道自己想要“张惠妹那种主歌隐忍、副歌爆炸”，但不会用英文系统地写出来
* 写出来的 Styles 要么太短、信息太少，要么东一块西一块，没有结构
* 不知道哪些词是 Suno 比较“听得懂”的编曲/动态词汇

结果就是模型发挥空间过大，同一首歌每次出来的结果都不太稳定。

### 痛点二：Lyrics 字段的“高级用法”很难落地

Suno 官方推荐在 Lyrics 里使用属性注释（如 `[Vocal: ...]`、`[Dynamics: ...]`），但普通创作者通常会卡在几件事上：

* 不知道应该有哪些 Key（Vocal / Dynamics / Texture / Mood / Arrangement / FX…）
* 不会用英文把“这一段主歌更克制、副歌和声更厚”这类感觉，拆成结构化的 `[Key: Value]`
* 试着写了几行注释，但要么和 Styles 描述矛盾，要么风格不统一

最终，大部分人会退回到最简单的用法：Lyrics 字段只放原文歌词，完全浪费了 Suno 支持的属性体系。

### 痛点三：歌词不敢随便交给 AI 改写

很多 LLM 工具默认会“帮你润色”歌词文本，这对“翻唱原曲”的场景来说基本是灾难：

* 不能让 AI 动原词，也不能多删几句，否则就不是那首歌了
* 创作者其实希望的是：只要帮我写 Styles 和属性注释，歌词一个字都不要给我动

### 痛点四：段落结构标签混乱

真实使用时，大家会写各种不同的段落标签：
* 【主歌】/【主歌1】/【主歌A】
* [verse] / [Verse] / [主歌]
* 【副歌】/ [chorus] / [Chorus]
* 【桥段】/ [Bridge]
* 【前奏】/【间奏】/【尾奏】

而 Suno 官方推荐的是标准英文标签：[Verse 1]、[Chorus]、[Bridge]、[Intro]、[Interlude]、[Outro]…

手工去统一一遍段落命名，本身就是一件费时又枯燥的事情。

## 核心功能

1. **智能生成Styles**：根据用户的中文风格描述，自动生成符合Suno官方术语体系的高质量英文Styles
2. **结构化Lyrics属性**：自动识别并标准化歌词段落标签，为每段歌词生成结构化的属性注释
3. **严格保护歌词原文**：只在歌词前添加工程标注，不修改、不润色歌词主体内容
4. **纯前端架构**：无后端、不存库、不打日志，完全保护用户隐私

## 适用人群

* 希望提升Suno翻唱质量的重度用户
* 懂音乐但英文表达一般的创作者
* 想固化风格策略的音乐人和团队
* 制作系列翻唱内容的创作者

## 快速开始

### 1. 安装依赖

```bash
pnpm install
```

### 2. 启动开发服务器

```bash
pnpm dev
```

应用将在 `http://localhost:8000` 启动。

### 3. 构建生产版本

```bash
pnpm build
```

构建产物将生成在 `dist` 目录中。

### 4. 部署

```bash
pnpm deploy
```

## 功能使用指南

### 基本流程

1. **输入中文风格描述**：在页面的输入框中用中文描述您想要的音乐风格和编曲想法
2. **输入歌词**：在歌词输入框中粘贴或输入您要翻唱的歌词
3. **生成提示词**：点击"生成提示词"按钮，系统将自动生成符合Suno官方术语体系的英文提示词
4. **复制使用**：将生成的提示词复制到Suno对应的输入框中使用

### 高级功能

* **段落标签标准化**：系统会自动识别并标准化您输入的歌词段落标签
* **自定义属性**：您可以根据需要自定义歌词的属性注释
* **历史记录**：系统会保存您的历史生成记录，方便您查看和复用

## 部署说明


2. 检查构建产物是否在 `dist` 目录中

将 `dist` 目录中的所有文件上传到您的服务器或托管平台即可。
